Lab 1 Test Cases: Hidden Functionality and Improper Authorization

Purpose:
These test cases verify the behavior of the vulnerable and fixed versions of the Lab 1 application. They are designed to show normal authentication behavior, exploitation of the hidden functionality, and confirmation that the vulnerability has been removed after remediation.

Test Environment:
- Platform: Ubuntu virtual machine
- Language: Python 3
- Application Files:
  - vulnerable_app.py
  - fixed_app.py

--------------------------------------------------
Test Case 1: Valid Standard User Login
--------------------------------------------------
Test ID: L1-TC-01
Objective:
Verify that a standard user can log in successfully and access only non-admin functionality.

Input:
Username: alice
Password: alice123

Steps:
1. Run the application.
2. Enter username "alice".
3. Enter password "alice123".
4. Select option 1 to view profile.
5. Select option 2 to attempt admin panel access.

Expected Result:
- Login is successful.
- User role is displayed as "user".
- Profile can be viewed.
- Admin panel access is denied.

Test Case 5: Hidden Functionality Exploit in Vulnerable Version
--------------------------------------------------
Test ID: L1-TC-05
Objective:
Demonstrate that the vulnerable application grants unauthorized admin access through the hidden password.

Input:
Username: testuser
Password: letmein

Steps:
1. Run vulnerable_app.py.
2. Enter username "testuser".
3. Enter password "letmein".
4. Select option 2 to access the admin panel.

Expected Result:
- Login should not succeed because the username is invalid.

Actual Vulnerable Behavior:

Summary of Test Coverage
--------------------------------------------------
These test cases validate:
- Successful standard user authentication
- Successful admin authentication
- Rejection of invalid credentials
- Presence of the hidden backdoor in the vulnerable version
- Removal of the hidden backdoor in the fixed version
- Proper authorization enforcement for admin-only functionality
- Basic application flow and usability
